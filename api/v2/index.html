List Executions

List Executions retrieves Automation Config Executions subject to the provided filters.

Authorizations:
API_Keys
query Parameters
pagination.page
integer <int32>

Page is the page to view. Pages use one-based indexing so the first page is always page 1.

pagination.per_page
integer <int32>

PerPage is the number of items per page.

order_by
string

OrderBy is a comma separated value to define sorting. e.g 'name,created_time DESC'. The values are separated by space, '{column} {ASC/DESC}'. These are the values that are sortable: none

filter
string

Filter is a string containing logic that can specify a restricted set of Recordsets to be returned. e.g. name co 'Test' or name co 'Acme'. These are the supported fields for filtering: none

Create Automation Config

Create Automation Config validates and creates an Automation Config based on the provided ordered tasks.

Authorizations:
API_Keys
Request Body schema: application/json

CreateConfigRequest is the information provided when creating an Automation Config.

cloud_vault_id
string

The ID of the Cloud Vault to be used for reading and writing data during Automation Execution.

cron_timer
string

The execution schedule of the Automation in UNIX CRON format as described in https://man7.org/linux/man-pages/man5/crontab.5.html

description
string

The user-friendly description of the Automation Config.

enabled
boolean

The 'enabled' flag allows Automation Execution triggers to be paused and resumed without adjusting the rest of the config or schedule.

end_date_time
string <date-time>

Optional UTC date-time to cease triggering Automation Executions. Format is ISO 8601 e.g. 2019-11-23T18:30:00Z

name
string

The user-friendly name of the Automation Config.

start_date_time
string <date-time>

The UTC date-time to begin triggering Automation Executions based on the CRON schedule. Format is ISO 8601 e.g. 2019-11-23T18:30:00Z

Array of objects (Automation_Configs.AutomationStep)

The sequence of tasks which constitute an Execution of this Automation Config.

timezone_identifier
string

The Timezone Identifier to schedule Automation Executions relative to. The format is form the IANA TZ database e.g. 'Europe/London' or 'America/New_York' from: https://en.wikipedia.org/wiki/List_of_tz_database_time_zones

Responses

https://infosum.github.io/api/v2/automations/configs
